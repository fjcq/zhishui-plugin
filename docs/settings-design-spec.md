# 止水插件设置面板设计规范

## 一、设计概述

### 1.1 设计目标

解决现有设置面板存在的三大核心问题：
1. **分类逻辑混乱** - 用户心智模型与实际结构不匹配
2. **信息层级扁平** - 缺乏优先级区分，高频操作难以快速定位
3. **操作便捷性不足** - 学习成本高，缺少引导

### 1.2 设计原则

| 原则 | 说明 |
|-----|------|
| 功能聚合 | 相关设置项按功能模块组织，减少用户跨分组操作 |
| 优先级分明 | 必填项、高频操作置顶，提供快速配置入口 |
| 渐进式披露 | 基础设置优先展示，高级设置折叠隐藏 |
| 一致性体验 | 统一组件样式、交互模式和文案风格 |

## 四、交互规范

### 4.1 表单验证

| 验证类型 | 触发时机 | 反馈方式 |
|---------|---------|---------|
| 必填验证 | 提交时 | 红色边框 + 错误提示 |
| 格式验证 | 失焦时 | 实时提示 |
| 范围验证 | 输入时 | 限制输入范围 |

### 4.2 状态反馈

```
保存成功：显示成功Toast，3秒后自动消失
保存失败：显示错误Toast，需手动关闭
加载中：显示Loading状态，禁用操作按钮
```

### 4.3 操作确认

| 操作类型 | 确认方式 |
|---------|---------|
| 删除配置 | 弹窗确认 |
| 重置配置 | 弹窗确认 |
| 保存配置 | 自动保存或显式保存按钮 |

## 五、文案规范

### 5.1 标签命名

- 使用简洁明了的中文
- 避免技术术语，使用用户易懂的表达
- 保持一致性（如：不混用"开启"和"启用"）

### 5.2 帮助文本

- 说明"为什么"而非"是什么"
- 提供示例值或格式说明
- 标注必填项和可选项

**示例：**
```
✅ 好：API密钥，如 sk-xxxx（必填）
❌ 差：请输入API密钥

✅ 好：最多保存几条对话记录，建议10-20条
❌ 差：最大历史记录数
```

### 5.3 占位符文本

- 提供示例值或格式提示
- 使用灰色文字（`#999`）
- 不重复标签内容

## 六、组件使用指南

### 6.1 SOFT_GROUP_BEGIN

用于创建分组容器，配合图标使用：

```javascript
{
    label: '⚡ 快速配置',
    component: 'SOFT_GROUP_BEGIN'
}
```

### 6.2 GSubForm

用于嵌套表单，支持多项目和单项目模式：

```javascript
{
    field: 'chat.ApiList',
    label: 'API配置',
    component: 'GSubForm',
    componentProps: {
        multiple: true,           // 多项目模式
        addButtonText: '添加配置',
        modalTitle: '编辑配置',
        schemas: [...]            // 子表单Schema
    }
}
```

### 6.3 动态属性

使用 `dynamicProps` 和函数形式的 `componentProps` 实现动态行为：

```javascript
{
    field: 'title',
    label: '角色标题',
    component: 'Input',
    componentProps: {
        disabled: (formData) => {
            // 根据表单数据动态禁用
            return formData._isDefault === true;
        }
    }
}
```
